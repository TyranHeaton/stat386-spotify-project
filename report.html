Spotify Music Analysis: Exploring Artist Consistency and Song Similarity

Introduction

Music is a uniquely measurable art form. Through the Spotify API, nearly every song can be quantified by audio features such as tempo, energy, acousticness, and valence. This project explores a dataset of approximately 170,000 Spotify songs to answer the following core questions:

  • How consistent are artists in their sound across their catalog?
  • How have key audio features changed over time?
  • Can we build a tool to recommend similar songs based on audio characteristics?

The result is a Python package (spotify_tools) and an interactive Streamlit web application that allows users to explore these questions visually and interactively.

Data

The dataset was obtained through the Spotify API prior to Spotify’s security enhancements in 2024. It contains approximately 170,000 songs and includes the following audio features used in this analysis:

Feature Description
duration_s Song length in seconds
tempo Beats per minute
acousticness Confidence measure of acoustic sound (0–1)
energy Perceptual intensity and activity (0–1)
valence Musical positivity (0–1)
popularity Spotify popularity score (0–100)
explicit Whether the song contains explicit content

Data Cleaning

The raw dataset required several cleaning steps before analysis could begin.

Separating Artists

In the raw data, the artists column stores multiple artists as a Python list within each row, to account for collaborations and features. The fix_artists() function expands this list so each artist gets its own row, making grouping and filtering by artist possible.

from spotify_tools.clean import fix_artists

df = fix_artists(df)

Filtering the Artist List

With 170,000 songs, the dataset includes many artists with only a handful of songs or a very narrow date range. These do not work well for meaningful individual artist analysis. The reduce_list() function applies three filters to address this:

  1. Removes artists with fewer than 40 songs
  2. Removes artists active for fewer than 10 unique years
  3. Removes artists with a high average acousticness (≥ 0.75), which filters out classical and instrumental catalogs
from spotify_tools.clean import reduce_list

df = reduce_list(df)

Column Standardization

Song duration is converted from milliseconds to seconds, and the explicit column is converted from binary integers (1/0) to readable labels (Yes/No). The dataset is then sorted, trimmed to the relevant columns, and deduplicated.

from spotify_tools.clean import column_cleaning

df = column_cleaning(df)

df = df.sort_values(by='artists').reset_index(drop=True)
df = df[['name', 'artists', 'year', 'duration_s', 'tempo', 'acousticness',
         'energy', 'valence', 'key', 'popularity', 'explicit']]
df = df.drop_duplicates(subset=['name', 'artists']).reset_index(drop=True)

Analysis

Artist Summary Statistics

The first form of analysis is a per-artist summary. The artist_summary() function returns descriptive statistics for an artist’s key audio features, providing a quick snapshot of an artist’s average sound and range.

from spotify_tools.analysis import artist_summary

artist_summary(df, artist="Drake")

Artist Variability

A central question of this project is: how consistent is an artist’s sound? The song_variability() function answers this by calculating the standard deviation of five key features for each artist and averaging them into a single overall variability score. A high score indicates an artist who changes their sound frequently; a low score indicates a more uniform catalog.

from spotify_tools.analysis import song_variability

song_variability(df)

Feature Trends Over Time

Music itself changes over time. The feature_trend() function groups the dataset by year and calculates the mean of a selected feature for each year, making it easy to observe how aspects like tempo or acousticness have shifted across decades.

from spotify_tools.analysis import feature_trend

feature_trend(df, feature="energy")

Top Songs by Feature

The top_songs() function accepts any audio feature and returns the top-ranked songs for that feature, useful for identifying the loudest, fastest, or most acoustic tracks in the dataset.

from spotify_tools.analysis import top_songs

top_songs(df, feature="tempo", n=10)

Feature Correlations

To understand how audio features relate to one another, the feature_correlations() function generates a correlation matrix across the five primary features. This reveals relationships such as the well-known negative correlation between acousticness and energy.

from spotify_tools.analysis import feature_correlations

feature_correlations(df)

Song Recommendation

The recommend_similar() function provides a simple but effective recommendation engine. Given a song name, it uses cosine similarity to find the n most similar songs in the dataset based on four audio features: energy, acousticness, tempo, and valence.

Cosine similarity measures the angle between two feature vectors in a multidimensional space. Songs with a small angle between their feature vectors are treated as more similar, regardless of the magnitude of their feature values.

from spotify_tools.recommend import recommend_similar

recommend_similar(df, song_name="God's Plan", n=5)

Interactive App

The project is accompanied by an interactive Streamlit web application that makes these analyses accessible without writing any code.

App Features

The app opens with a dropdown menu containing every artist in the cleaned dataset, plus an All Artists option.

All Artists view: Displays the full dataset and provides three tabs — Length, Acousticness, and Tempo — each showing a scatterplot of that feature’s yearly average and a histogram of its overall distribution.

Individual Artist view: Filters the dataset to the selected artist and displays their songs in a table. The same three tabs appear, now showing that artist’s specific trends and distributions. Additionally, the user can select any of that artist’s songs and click Recommend to get five similar songs from across the entire dataset, powered by the cosine similarity recommendation function.

Conclusion

This project demonstrates that audio features from the Spotify API can be used to meaningfully characterize and compare artists, track how music has evolved over time, and identify similar songs. The custom Python package (spotify_tools) packages this analysis into reusable functions, while the Streamlit app makes the results accessible to anyone.

Future directions could include:

  • Adding more audio features to the recommendation engine (e.g., danceability, loudness)
  • Clustering artists by audio profile
  • Comparing different time periods or music genres
  • Using a larger, continuously updated dataset via a re-authorized Spotify API connection
